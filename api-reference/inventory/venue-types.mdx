# Venue Types API

Venue types classify where screens are physically placed. They form a three-level hierarchy used for ad targeting ГўВЂВ” advertisers select campaigns based on venue types, so accurate classification directly affects which ads your screens receive and how much revenue they generate.

## The Venue Type object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique venue type identifier |
| `label` | string | Human-readable name (e.g., "Airport", "Shopping Mall") |
| `parent_id` | string \| null | ID of the parent venue type. `null` for level 1 entries. |
| `level` | int | Hierarchy level: `1`, `2`, or `3` |
| `selectable` | boolean | Whether this type can be assigned to a location or screen |
| `description` | string \| null | Explanation of what this venue type represents |

## Hierarchy levels

| Level | Selectable | Assign to | Example |
|-------|-----------|-----------|---------|
| 1 | No | ГўВЂВ” | "Transit", "Retail", "Hospitality" |
| 2 | Yes | **Locations** | "Airport", "Shopping Mall", "Hotel" |
| 3 | Yes | **Screens** | "Arrival Hall (Airport)", "Food Court (Mall)" |

Level 1 entries are category groups ГўВЂВ” they exist only for organizational purposes and cannot be assigned to locations or screens. When creating a location, use a **level 2** venue type. When creating a screen, use either the **same level 2** type as the parent location, or one of that type's **level 3 children**.

### Parent-child constraint

A screen's venue type must be related to its parent location's venue type:

```
Location: venue_type_id = "101" (Airport, level 2)
  ГўВ”В”ГўВ”ВЂГўВ”ВЂ Screen: venue_type_id = "101"    ГўВњВ… Same as location (level 2)
  ГўВ”В”ГўВ”ВЂГўВ”ВЂ Screen: venue_type_id = "10101"  ГўВњВ… Child of "101" (level 3)
  ГўВ”В”ГўВ”ВЂГўВ”ВЂ Screen: venue_type_id = "501"    ГўВќВЊ Unrelated venue type
```

You can verify relationships using the `parent_id` field: a level 3 entry's `parent_id` will match the `id` of its level 2 parent.

See [Data Formats ГўВЂВ” Venue Type Hierarchy](/data-formats#venue-type-hierarchy) for more details on the ID structure.

## List all venue types

```
GET /vendor/{org_token}/venue_types
```

This is a read-only endpoint. Venue types are managed by FRAMEN and cannot be created or modified through the API.

### Path parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `org_token` | string | Yes | Your organization token |

### Example request

```bash
curl -X GET "https://api.framen.com/vendor/Org_9WeJxcVrRnM/venue_types"
```

Returns an array of all venue types in the taxonomy. There is no pagination or filtering ГўВЂВ” the full taxonomy is returned in a single response. Cache this response and refresh periodically (e.g., daily) rather than fetching it on every operation.
