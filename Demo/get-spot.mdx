# Get Spot

The Vendor Default API lets you request a single ad creative for a specific screen, then confirm delivery via a **Proof of Play** (PoP) callback.

<Warning>
The Proof of Play URL returned in each response must be fired within **1 hour** of the request. It can only be registered **once**.
</Warning>

---

## Get Spot

<api-endpoint method="GET" url="https://api.framen.com/v3/vendor/{org_token}/1/rt_ad/{screen_id}" />

Requests a creative for a single playout on the specified screen.

## Proof of Play

Every content item in the response includes a `pop` field — a GET URL that **must be called exactly once** after the item has been displayed.

```
GET {pop_url}
```

<Warning>
- Fire the PoP URL **within 1 hour** of the original ad request
- Each PoP URL can only be registered **once** — duplicate calls are ignored
- Failing to fire PoP means the playout is not confirmed and may affect billing
</Warning>

---

## Request Cycle

The typical integration loop works like this:

<Steps>
  <Step title="Pre-cache: Request Ad">
    Call `GET /rt_ad/{screen_id}` ahead of the scheduled slot to pre-fetch the creative.
  </Step>
  <Step title="Display the Creative">
    Render the `url` from `data.ad` or `data.news` on screen for the specified `duration`.
  </Step>
  <Step title="Fire Proof of Play">
    Immediately after display, call the `pop` URL once via GET. Must happen within 1 hour of the original request.
  </Step>
  <Step title="Repeat">
    Request the next creative for the following slot, overlapping pre-cache with the current playout.
  </Step>
</Steps>
