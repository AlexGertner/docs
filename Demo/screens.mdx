# Screens

Screens represent individual display units within a Location. They carry the technical specifications and audience data that power ad targeting and delivery.

<Info>
Screens cannot exist without a Location. Always [create a Location](/api-reference/inventory/locations) first.
</Info>

<Warning>
`screen.venue_type_id` must be the same as вАФ or a child of вАФ `location.venue_type_id`. Mismatches will be rejected.
</Warning>

---

## Create Screen

<api-endpoint method="POST" url="https://api.framen.com/vendor/{org_token}/screens" />

Creates a new Screen within your organization.

| Parameter | Type | Required | Example |
|-----------|------|----------|---------|
| `org_token` | string | вЬЕ | `Org_9WeJxcVrRnM` |

### Body Parameters

| Parameter | Type | Required | Example | Notes |

| `screen_id` | string | вЬЕ | `d3c8dbb6-c21b-4b26-8f35-d73260c95fa1` | Your internal screen GUID |

| `venue_type_id` | enum | вЬЕ | `7FR-147067ae-e630-46da-b31a-56dad634ec4a` | Must match or be child of location's venue type |

| `width` | int | вЬЕ | `1920` | Pixels |
| `height` | int | вЬЕ | `1080` | Pixels |
| `loop_duration` | int | вЬЕ | `120` | In seconds |
| `archived` | boolean | вАФ | `false` | When `true`, screen disappears from Dashboard UI |
